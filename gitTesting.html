- git 工作；流程 【工作区 workding directory】 -> 【暂存区 Staged】 -> 【完成区 commited】 1. Set global command git config --global user.name 'Pan Chen' git config --global user.email 'pc87031156@gmail.com' 2. Get help git 'verb' -h 3. initalize a normal folder to git hub folder -> 初始化git目录 git init 4. git status -> 显示文件状态 - 简写 ： git status -s / git status --short -> 结果如果是以 ?? 开头的 表示该文件并没有存入 git 目录中 5. git add 'GitTesting.html' -> 将该文件跟踪加入进 缓存区 - 使用 git status 重新检测 - 简写 ： git status -s / git status --short -> 结果是新的文件 A 的话 证明该文件已经加入进 【缓存区】 中 6. git commit -> 将缓存区的文件提交至 【完成区】 - git commit -m 'Add GitTesting html file' - 使用 git status 重新检测 - 简写 ： git status -s / git status --short -> 结果是 M 蓝色的话 证明该文件已经加入进 【完成区】 中 7. 如果文件在 【完成区】中 ， 但是在工作区中修改了文件 此时执行 git status / git status -s - 使用 git status 查看状态 - git status -s 结果是以 M 红色开头的话 ， 那么就是 【工作区】中的文件与【完成区】文件不一致 8. 执行 撤销操作 - 在 git commited -m 'message' 之后， git 仓库 和 【工作区】 文件就是一致的 ， 那么如果此时去修改 【工作区】的文件 ，git status 之后就变成了 M 红 - 那么如果想要撤销本次的操作， 回归于之前 git 仓库的文件 命令如下啊 - git checkout -- 文件名 -> 一旦执行此命令当前修改就没有了 9. 一次提交多个文件进入【暂缓区】 - git add . -> 执行此命令后将会把当前目录中 新添加的 或者修改 都会加入进暂缓区中 10. 删除暂缓区的文件 - git reset HEAD 文件 - git reset HEAD . -> 删除所有暂存区的文件 11. 【工作区】 -> 【暂缓区】 -> 【Git仓库】 - 此时执行以下命令可以直接从 【工作区】 -> 【Git仓库】 - git commit -a -m '跳过了 暂缓区直接提交进Git仓库' 12. 删除文件 - git rm -f 文件名 -> 这里将会删除 【Git仓库】和 【工作区】中的文件 【此时如果使用 git status -s 查看状态的话 就是 D 绿色】 - git rm -cached 文件名 -> 这里将只会删除 【Git仓库】中的文件 13. 忽略指定文件 - 首先是要 touch 一个名为 .gitignore 的文件 ，在里面添加需要被忽略的文件 14. 查看 log - git log -> 所有Log -> git log -10 -> 前10条 log 15. 使用 log id ,回溯至指定位置 - git log --pretty=oneline -> 以一条的方式展示 - git reset --hard "log id" - 如果回溯错误 ， 取之前的 log 再回溯回来 - git reflog
